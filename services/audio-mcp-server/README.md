# Audio MCP Server

MCP server responsible for audio generation tools.

The server runs over MCP STDIO and exposes:

- `generate_audio_from_text`
- `save_audio_file`
- `get_audio_metadata`

## Role In The System

The Audio MCP Server receives text and returns a generated audio file. It does not know whether the text came from:

- a podcast script generated by the internal Script Agent
- direct `read_aloud` narration
- another future text preparation flow

That decision belongs to the API Host.

## Response Shape

`generate_audio_from_text` returns:

```json
{
  "audio_url": "/generated/audio/podcast-test.wav",
  "format": "wav",
  "duration_seconds": 120
}
```

Generated files are written under `generated/audio/`.

## TTS Provider Layer

Audio synthesis is isolated behind `TtsProvider`:

```txt
Audio MCP tool
  |
TtsProvider
  |-- MockTtsProvider
  `-- OpenAiTtsProvider
```

Provider selection is controlled by `.env`:

```env
TTS_PROVIDER=mock
OPENAI_API_KEY=
OPENAI_TTS_MODEL=tts-1
OPENAI_TTS_VOICE=coral
OPENAI_TTS_RESPONSE_FORMAT=wav
OPENAI_TTS_INSTRUCTIONS=Speak clearly in a warm podcast narration style.
```

Use `TTS_PROVIDER=mock` for deterministic local development without API calls.

Use `TTS_PROVIDER=openai` for real speech generation through OpenAI.

Supported language values passed by the API Host:

- `en` for English
- `es` for Spanish
- `pt` for Portuguese

## Run Directly

The API Host starts this MCP server over STDIO when needed. For debugging, it can also be run directly:

```bash
uv run python services/audio-mcp-server/src/audio_mcp_server/server.py
```
